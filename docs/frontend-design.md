# Aurora 前端设计与架构（aurora-ui）

状态：v0.1 草案（2026-07-13）。技术栈 iced 0.14 原生 GUI。本文是前端施工蓝图，设计决策来自与用户的两轮确认。动效弹簧参数一节待 mailauncher dev 分支调研回填。

## 一、设计系统

### 色彩
- 双主题：亮色为主 + 暗色，两套都做，运行时可切换（可跟随系统）。
- 强调色：浅蓝 -> 粉 的约 45 度斜向渐变。用于按钮、进度、高光、选中态、焦点环。
- 无彩体系：白/浅底（亮）与深底（暗），中性层级走灰阶梯度。
- 语义色（成功/警告/错误）最小化，尽量并入强调色体系，不引入杂色。
- 令牌化：所有颜色/渐变以主题令牌暴露，页面与组件只引用令牌，不写死色值。

### 材质
- 窗口：无边框 + 圆角 + 亚克力/云母背景。
- 内容卡片：浮于背景之上的半透明毛玻璃。
- 亚克力在老 Win10/精简系统上会失效 -> 自动降级为绘制的极光渐变背景（见下）。
- 注：mailauncher 的 Living Surfaces 走“暖哑光面、零玻璃”路线，与 Aurora 选定的亚克力毛玻璃是两种质感。Aurora 只借用其弹簧手感，表面材质按亚克力走（除非另行决定改哑光）。

### 排版
- 中文默认走系统字体（微软雅黑 / Segoe UI 混排），最稳、不增打包体积。可后续换思源黑体提升质感（代价：体积）。
- 字号/行高/字重令牌化。

### 图标
- 细线描边（line）。首选 Lucide（线性、开源 ISC），SVG 内联，描边色跟随主题/强调色。

## 二、布局与窗口

- 主导航：左侧可展开图标栏。收起态=纯图标；展开态=图标+文字（hover 或点击展开，展开动画走弹簧）。
- 顶部：自绘标题栏（窗口拖拽、最小化、关闭），配合窗口圆角。
- 右侧：内容区，承载各页面。
- 背景：默认内置蓝粉极光动态背景（缓慢流动）；同时支持用户换成自己的图。极光背景亦是亚克力失效时的降级方案。

## 三、动效

总基调：弹性（spring），华丽游戏化。三类核心行为：
1. 弹簧（两参模型，非“二段”）：源自 mailauncher develop 分支 frontend/src/design/motion.ts 的 Living Surfaces 动效系统。术语更正——是“两参弹簧”（用感知时长 duration + 弹跳量 bounce 两个参数刻画的标准阻尼谐振子，HyperOS/iOS 风），不是两段/分段弹簧。换算：ω=2π/duration，刚度 k=ω²，阻尼 c=2(1−bounce)ω，质量 m=1，阻尼比 ζ=1−bounce。五枚预设（k/c 已换算，mass=1）：
   - springTap    duration0.22 bounce0.00  k=815.6 c=57.12  ζ=1.00  按压/悬停（零过冲，脆）
   - springSettle duration0.26 bounce0.10  k=584.0 c=43.50  ζ=0.90  卡片落定/滑块跟随（最常用）
   - springPop    duration0.32 bounce0.20  k=385.5 c=31.42  ζ=0.80  原位往返归位（过冲最明显）
   - springSoft   duration0.34 bounce0.12  k=341.5 c=32.53  ζ=0.88  整页入场
   - springMorph  duration0.35 bounce0.14  k=322.3 c=30.88  ζ=0.86  图标↔菜单形变
   实现：Spring{current,velocity,target,k,c,mass}，半隐式欧拉逐帧 step，settled 阈值约 0.01 后吸附并停订阅；目标变化只改 target 不清零 velocity（可中断、有惯性）。springTap 的 ζ=1 不加过冲。
2. 原位往返（共享元素过渡）：元素从触发它的位置飞出，返回时飞回原位。需要一个 overlay 过渡层记录源/目标几何并驱动飞行插值。
3. 滑动卡片：列表/面板的滑入滑出与重排（FLIP 式）。

实现约束（iced 现实）：iced 无内建动画系统。统一用时间订阅（约 60fps 的 frame tick）驱动数值插值，`Message::Tick` 内用相邻时间戳算真实 dt（不假设固定 16ms）。每个动画量封装为弹簧状态（current/velocity/target/参数），收敛后停订阅省电；任一弹簧在动就继续订阅。

## 四、前端 crate 架构

新建 `crates/aurora-ui`（iced 0.14 二进制，依赖 aurora-core）。先立地基、再长页面：

1. `theme/` — 主题令牌（色/渐变/间距/圆角/阴影/字体）+ 亮暗双主题 + 运行时切换。
2. `anim/` — 动画内核：通用弹簧（二段弹簧）+ 时间订阅驱动的每帧插值 + 收敛判定；overlay 原位往返过渡层。
3. `widgets/` — 自绘组件：毛玻璃卡片、可展开导航项、滑动卡片、自绘标题栏、按钮，内建动效。
4. `platform/` — 窗口集成：无边框 + 圆角 + 亚克力（window-vibrancy），含低配降级检测与极光背景回退。
5. `pages/` — 各页面（主页/账户/版本/下载/设置…），在地基上搭，经 aurora-core 接后端。
6. `app` — 顶层 iced 应用：状态、消息、订阅、路由与过渡编排。

依赖 API（iced 0.14、window-vibrancy、原生窗口句柄获取等）落笔前一律经 Context7 核对当前版本，不凭记忆写。

## 五、第一里程碑（先做，尽早验收手感）

应用外壳（自绘标题栏 + 亚克力窗口 + 左侧可展开导航 + 极光背景）+ 主题系统（亮暗双主题 + 蓝粉渐变令牌）+ 二段弹簧动画内核 + 一个示范页：用导航切换演示原位往返过渡与滑动卡片。手感确认后，再填接 aurora-core 的真实页面。

## 六、默认项（未被推翻则采用）

- 图标集：Lucide。
- UI 字体：系统微软雅黑 / Segoe UI 混排。
- 默认窗口：无边框圆角，约 1000x640，可缩放并设最小尺寸。

## 七、技术风险与对策

- 亚克力需拿原生窗口句柄交给 window-vibrancy；iced 0.14 的确切 API 落笔时 Context7 核对。亚克力在老系统失效 -> 极光渐变背景兜底（用户本就要这个背景，观感不受损）。
- 动效全手写，工期长于 Web/Slint 方案；以第一里程碑先立动画基础设施来控制风险。
